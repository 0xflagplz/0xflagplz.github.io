Anything with a gets()will be succeptable to a buffer overflow (unless a canary is active which will shut it down)
So lets look at a simple buffer overflow example.

#include <stdlib.h>
#include <unistd.h>
#include <stdio.h>
#include <string.h>

int main(int argc, char **argv)
{
char buffer[55];

gets(buffer);
}

How do we know where the overflow starts?
  • > We look to get a segmentation fault (i.e. we run out of memory for our input)
